# AGENTS.md Template

## Purpose

This template creates an AGENTS.md file following the [open AGENTS.md format](https://agents.md/) — a standard Markdown file that helps AI coding agents understand and work with the project. Think of it as a README for agents.

## Template Structure

```markdown
# AGENTS.md

This project specification was generated by the Design Agent workflow. Below is the context and guidance AI coding agents need to implement this project effectively.

## Project overview

[Project Name] — [1-2 sentence project description extracted from requirements].

Generated on: [ISO 8601 Date]

## Required reading

Read these files first to understand the project requirements and architecture. All paths are relative to the `generated/design/` directory.

- `requirements/functional-requirements.md` — Functional requirements with acceptance criteria
- `requirements/non-functional-requirements.md` — Non-functional requirements (performance, security, scalability)
- `architecture/system-architecture.md` — System architecture with component specifications and diagrams

## Recommended reading

These files provide additional context for implementation decisions.

- `security/threat-analysis.md` — STRIDE-based threat identification and analysis
- `security/security-controls.md` — Security implementation requirements and controls
- `architecture/architecture-decision-records/` — Architecture Decision Records (ADRs) documenting key decisions

## Code style

[Add project-specific code style guidelines derived from requirements, e.g.:]

- Follow the technology stack and patterns defined in `architecture/system-architecture.md`
- Refer to ADRs for rationale behind architectural choices

## Testing instructions

[Add project-specific testing guidance derived from requirements, e.g.:]

- Acceptance criteria for each requirement are defined in `requirements/functional-requirements.md`
- Security testing requirements are in `security/security-controls.md`

## Important notes

- This specification was generated by automated analysis and requires human validation
- See `README.md` for human-readable navigation and stakeholder guidance
- See `project-management/executive-summary.md` for business context
```
